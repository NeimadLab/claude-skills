# Gateway Model

## Overview

The MCP Gateway is Loom's central routing layer. It serves both local and remote clients using the standard Model Context Protocol. Every model interaction — whether from Claude Code on your laptop or Claude.ai in the cloud — passes through the gateway's authentication, policy, and audit pipeline.

## Transports

| Transport | When | Configuration | Security |
|-----------|------|--------------|----------|
| **stdio** | Local clients (Claude Code, Cursor, Windsurf) | Default. Zero config. Auto-discovered via `loom connect`. | Inherently trusted (local user). |
| **SSE/HTTPS** | Remote clients (Claude.ai, ChatGPT, CI/CD) | Opt-in. Requires auth config in `.loom/policies/`. | TLS 1.3 required. API key or OAuth. |

## MCP Tool Surface

These are the MCP tools exposed by the gateway. The same tools are available in both local and remote topologies — the difference is the default policy scope.

| Tool | Tier | Description | Local Default | Remote Default |
|------|------|-------------|:------------:|:--------------:|
| `loom_get_context` | Read | Compact project onboarding package | Allow | Allow |
| `loom_get_handoff_summary` | Read | Structured transfer packet for model onboarding | Allow | Allow |
| `loom_search_memory` | Read | Full-text + metadata memory search | Allow | Allow |
| `loom_write_memory` | Write | Persist a structured memory entry | Allow | Writer role |
| `loom_log_decision` | Write | Record decision with rationale and status | Allow | Writer role |
| `loom_list_files` | Read | Browse project file inventory | Allow | Allow (filtered) |
| `loom_read_file` | Read+ | Read file content (sensitive paths blocked) | Allow | Approve |
| `loom_write_file` | Write+ | Write file content (requires write token) | Writer token | Approve |
| `loom_execute` | Execute | Run command in workspace (highest privilege) | Approve | Deny (default) |
| `loom_read_secret` | Secret | Access named secret alias | Whitelist | Deny (always) |

## Request Pipeline

Every MCP tool call follows this pipeline:

```
Client sends MCP tool call
  → Transport layer (stdio or SSE) receives request
  → Authentication: verify client identity (API key, OAuth token, or local trust)
  → Session validation: verify active session with valid lease
  → Policy evaluation: check .loom/policies/*.yaml for matching rule
    → Allow: proceed to dispatch
    → Deny: return error with reason, log event
    → Approve: prompt user for approval (CLI or webhook), wait for response
  → Dispatch: route to appropriate Loom core function (P1, P2, or P3)
  → Capture result
  → Audit: log to events.jsonl (who, what, when, decision, outcome, duration)
  → Return result to client
```

## Remote Deployment

### Docker Compose

```yaml
services:
  loom:
    image: ghcr.io/neimadlab/loom-server:latest
    ports:
      - "8443:8443"
    volumes:
      - loom-data:/workspace/.loom
      - ./project:/workspace
    environment:
      - OAF_API_KEY=${OAF_API_KEY}
      - OAF_TRANSPORT=sse
      - OAF_LOG_LEVEL=info
    restart: unless-stopped

volumes:
  loom-data:
```

### Fly.io

```toml
# fly.toml
app = "my-loom-instance"
primary_region = "cdg"

[build]
  dockerfile = "deploy/docker/Dockerfile"

[env]
  OAF_TRANSPORT = "sse"

[mounts]
  source = "loom_data"
  destination = "/workspace/.loom"

[[services]]
  internal_port = 8443
  protocol = "tcp"
  [[services.ports]]
    port = 443
    handlers = ["tls", "http"]
```

## Client Configuration

### Claude Code (local)

Generated by `loom connect claude-code`:

```json
{
  "mcpServers": {
    "loom": {
      "command": "loom",
      "args": ["mcp", "serve", "--transport", "stdio"]
    }
  }
}
```

### Claude Code (remote)

```json
{
  "mcpServers": {
    "loom": {
      "type": "sse",
      "url": "https://loom.myserver.com/mcp",
      "headers": {
        "Authorization": "Bearer ${OAF_API_KEY}"
      }
    }
  }
}
```

### Claude.ai Skill

The same remote configuration works as a Claude.ai MCP integration. Configure the SSE URL and API key in the skill settings. The MCP tools appear as available actions in the conversation.

## Session Management

- **Session creation:** automatic on first MCP tool call from a new client
- **Session identity:** ULID, linked to client identity and permissions scope
- **Lease duration:** configurable, default 30 minutes, renewable
- **Session expiration:** auto-expire after lease. Write token released. Event logged.
- **Forced close:** operator can force-close sessions via `loom gateway sessions --close <id>`
